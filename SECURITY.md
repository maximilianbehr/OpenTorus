# Security Policy

OpenTorus is **pre-alpha** software and should not be used in production.

## Supported versions

Only the latest `main` is supported during pre-alpha development.

## Reporting a vulnerability

If you discover a security issue, please **do not** open a public issue.
Instead, report it privately to the maintainers via GitHub's
["Report a vulnerability"](https://github.com/opentorus/opentorus/security/advisories/new)
workflow (Security → Advisories).

Please include:

- A description of the issue and its impact.
- Steps to reproduce, if possible.
- Any suggested remediation.

We will acknowledge your report as soon as we can and keep you informed as we
work on a fix.

## Privacy note

OpenTorus is local-first: project memory lives in `.opentorus/` on your
machine. Be mindful not to include secrets or sensitive data in artifacts you
share or attach to reports.
